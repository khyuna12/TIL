## 머신러닝(Machine Learning)
- 인간의 학습 능력을 컴퓨터로 실현하려는 기법
- 코딩하지 않고 예제를 통해 학습할 수 있는 시스템
- 경험을 통해 데이터 기반으로 학습하고 예측

<br>

### 머신러닝의 개요

<br>

#### 프로그래밍과 차이점
- 프로그래밍에서는 데이터와 규칙이 결합하여 출력 생성
- 머신러닝에서는 데이터와 출력이 함께 들어가서 규칙 생성

<br>

#### 머신러닝과 인공지능
- 머신러닝은 인공지능에 속하는 부분집합
- 인공지능
    - 추론
    - 자연어처리
    - 계획
    - 머신러닝
        - 지도학습
        - 비지도학습
        - 강화학습
        - 신경망(딥러닝)

- 머신러닝은 데이터로부터 학습하여 지식 획득
- 인공지능은 지식을 획득한 후, 그것을 활용함

<br>

#### 머신러닝의 주요 종류들

- **신경망(Neural Network)**: 생물의 신경 네트워크 구조와 기능을 모방한 모델
- **클러스터링(Clustering)**: 주어진 데이터를 클러스터라는 부분 집합들로 분리하는 것
- **분류(Classification)**: 주어진 데이터를 비슷한 것들끼리 분류하는 것
- **의사결정 트리(Decision Tree)**: 트리 구조 형태의 예측 모델로 의사를 결정하는 모델
- **나이브 베이즈(Naive Bayes)**: 베이즈 정리를 바탕으로 한 조건부 확률 모델 분류

<br>

#### 머신러닝의 활용 분야

- 영상인식
- 얼굴인식
- 음성인식
- 자연어 처리: 자동 번역, 대화 분석
- 정보검색: 스팸 메일 필터링
- 검색 엔진: 개인 맞춤식 추천 시스템
- 로보틱스: 자율주행 자동차, 경로 탐색

<br>

#### 머신러닝의 학습 방법

- **지도학습**
    - 주어진 입력과 정해진 출력 간의 관계를 학습
    - 각 데이터에 레이블 또는 태그 표시 붙임
    - 데이터에 P 또는 F 라는 레이블 활용
    - 이전의 경험으로부터 데이터 출력 생성
    - 출력에 반드시 레이블이 있는 데이터들을 사용해야 함
    - 계산 비교적 간단, 정확성 높음

- **비지도학습**
    - 출력값을 알려주지 않고 스스로 모델 구축하여 학습
    - 입력만 있고 출력 즉 레이블이 없음
    - 규칙성 스스로 찾아내야함
    - 데이터에 레이블을 전혀 사용하지 않음
    - 결과는 지도학습의 입력으로 사용 가능
    - 예: 비슷한 성향의 고객을 그룹으로 묶기, 네트워크상의 비정상적인 접근 탐지
    - 데이터 마이닝
    - 계산 복잡, 다소 덜 정확함

- **강화학습**
    - 주어진 입력에 대응하는 행동에 대해 보상, 이러한 보상을 이용해 학습
    - 주어진 입력에 대한 출력 즉 정답행동이 주어지지 않음
    - 입출력이 쌍으로 된 훈련 집합으로 제시되지 않는다는 점에서 일반적인 지도 학습과는 다름
    - 주요 응용분야: 로봇, 게임, 내비게이션, 체스와 바둑같은 게임

### **지도 학습**
> 분류, 회귀

- **분류**
    - 유사한 특성을 가진 데이터들끼리 묶어서 나누는 것
    - 이항 분류, 다항 분류
    - 예: 사진으로 남자 여자 구별, 개와 고양이 구분, 카드 부정 사용 감지, 의료 영상에서 종양의 존재 여부 판단
    - 방법
        - Naive Bayes 분류기
            - 자료의 분류를 베이즈 정리를 활용하여 판단
            - 조건부 확률 모델
            - 모든 특성값은 서로 독립이라고 가정
            - 장점: 구축하기 쉽고 대규모 데이터에 유용, 내용에 따라 분류 가능
        - 의사결정트리
            - 관측값과 목표값을 연결하는 예측 모델
            - 최대 2가지의 판단을 하는 이진 트리 사용
            - 스무고개 문답처럼 선택 방법
        - SVM(Support Vector Machine)
            - 데이터를 2개의 영역으로 분류하는 이진 분류기
            - 새로운 데이터가 어느 영역에 속하는지 판단
            - 가장 큰 폭을 가진 하나의 경게선을 찾는 알고리즘
            - 영역의 여백(margin, gap)이 최대가 되는 중심선 찾기
            - 패턴인식, 자료 분석
        - K-Nearest Neighbor(K-NN)
            - 최근접 이웃 분류
            - 가장 가까운 것들과의 거리 계산으로 클래스를 분류
            - 새로운 입력 데이터와 가장 가까운 k개의 이웃 데이터 선택
            - 이웃 데이터들의 클래스 중 다수결로 데이터의 클래스 결정
            - 다수결에서 결과가 나오기 위해 k는 반드시 홀수여야 함
            - 매우 간단하고 빠르고 효과적임, 어느 데이터라도 유사성 측정 가능
            - 적절한 k를 선택해야 하는 게 단점, 새로운 데이터에 대해 일일이 거리 게산 후 분류

- **회귀**
    - 변수들 사이의 관계를 결정하는 통계적 측정
    - 하나의 독립 변수를 사용하는 직선 형태의 '선형 회귀'
    - 선형 회귀는 각 점에서 회귀 직선까지 y축 방향의 거리(오차) 제곱의 총합을 최소로 해서 얻어지는 직선
    - y = a + bx를 x에 대한 y의 회귀 직선이라고 함
    - 예: 날씨에 대한 예측, 주택 가격 예측, 원유 가격 추정 등

- 회귀 분석
    - 변수 사이의 회귀에 대해 검정이나 추정을 하는 것
    - 학습 데이터를 사용하여 출력값 예측
    - 확률론적 의미를 내포

- 분류와 회귀의 차이점
    - 분류는 일정한 기준에 따라 명백하게 구분짓는 것
    - 회귀는 오차 제곱의 합을 최소화하는 직선을 긋는 작업
    - 분류의 출력은 선택식 출력
    - 회귀의 출력은 연속값

### **비지도학습**
> K-means 클러스터링, 가우스 혼합 모델, 계층적 클러스터링, 추천 시스템 등

- **클러스터링**
    - 클러스터: 유사한 여러 개의 클래스로 나누어진 데이터
    - 클러스터링: 유사한 특성을 가진 그룹들로 묶는 작업
    - 분류와 차이점
        - 분류는 데이터를 기준에 따라 직선으로 분류
        - 클러스터링은 유사성에 따라 몇 개의 클러스터들로 묶음

    - **K-means 클러스터링**
        - 유사한 특성을 가진 k개의 데이터 그룹으로 묶는 방법
        - k개의 클러스터 중심점 찾기
        - 장점: 알고리즘이 비교적 단순하고 수행속도가 빠름(주어진 데이터에 대한 사전 정보 없이 클러스터링을 함)
        - 단점: 클러스터링 개수 k와 최초로 지정하는 중심점들에 따라 결과가 다소 달라질 수도 있음

    - **추천 시스템**
        - 추천을 위해 연관 데이터 정의에 도움 주는 클러스터링 방법
        - 사용자의 '선호도'를 예측하는 정보 필터링의 일종

### **강화 학습**
> 베이지안 네트워크, 은닉 마르코프 모델

- **베이지안 네트워크**
    - 빌리프 네트워크
    - 집합을 조건부 독립으로 표현하는 확률의 그래픽 모델
    - 추론과 학습을 수행하기 위한 효과적인 알고리즘 존재
    - 예: 증상이 주어지면 다양한 질병의 존재 확률 계산 가능

- **은닉 마르코프 모델**
    - HMM
    - 은닉한 상태와 관측 가능한 결과로 이루어진 확률형 모델
    - 동적 베이지안 네트워크로 나타낼 수 있음
    - 대량의 데이터를 통계적으로 분석하여 추론에 응용
    - 음성인식, 자연어 처리 등
